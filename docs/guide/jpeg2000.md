# JPEG2000 Support

JPEG2000 (JP2 / J2K) is an advanced image compression standard that offers superior compression, lossless and lossy modes, multi-resolution representation, and support for high bit depths. `modern-pdf-lib` provides transparent decoding of JPEG2000-encoded images found in PDF files.

## Overview

JPEG2000 (ISO/IEC 15444 / ITU-T T.800) is commonly used in:

- **Medical imaging** (DICOM) — 16-bit grayscale, lossless compression
- **Satellite and geospatial** — large tiled images, multi-spectral bands
- **Government archives** — PDF/A-2 and PDF/A-3 allow JPEG2000 as a native image format
- **Digital cinema** (DCI) — 12-bit 4K/8K frames

PDF uses the **JPXDecode** filter to embed JPEG2000 images. Both raw J2K codestreams and full JP2-wrapped files are supported.

## Automatic Decoding

When you load a PDF containing JPEG2000-encoded image streams, `modern-pdf-lib` detects the `JPXDecode` filter and decodes the image transparently:

```ts
import { loadPdf } from 'modern-pdf-lib';

const pdf = await loadPdf(pdfBytes);
const pages = pdf.getPages();

// Images using JPXDecode are decoded automatically when accessed.
// No special configuration is required.
```

The decoder handles:

- Raw J2K codestreams (starting with `FF 4F` SOC marker)
- JP2-wrapped files (starting with the JP2 signature box)
- Single-tile and multi-tile images
- 1-bit through 16-bit component depths
- Grayscale, RGB, CMYK, and arbitrary component counts

## Manual Decoding

For advanced use cases, you can decode JPEG2000 data directly:

```ts
import { decodeJpeg2000 } from 'modern-pdf-lib';

const j2kBytes = new Uint8Array(/* ... */);

const decoded = decodeJpeg2000(j2kBytes);
// decoded.data     — Uint8Array of pixel data (interleaved components)
// decoded.width    — image width in pixels
// decoded.height   — image height in pixels
// decoded.components — number of color components
// decoded.bitsPerComponent — bit depth per component
```

### Extracting Tile Grid Information

For large images, you can inspect the tile grid before decoding:

```ts
import { parseTileInfo, decodeTile } from 'modern-pdf-lib';

const info = parseTileInfo(j2kBytes);
console.log(`Image: ${info.imageWidth}x${info.imageHeight}`);
console.log(`Tiles: ${info.tilesX}x${info.tilesY} (${info.tileWidth}x${info.tileHeight} each)`);

// Decode only the first tile
const tile = decodeTile(j2kBytes, 0);
console.log(`Tile 0: ${tile.width}x${tile.height} at (${tile.x}, ${tile.y})`);
```

## Multi-Resolution

JPEG2000 natively supports multi-resolution decoding through its wavelet decomposition. You can request a reduced-resolution version of the image for thumbnails or previews without decoding the full image:

```ts
import { decodeJpeg2000 } from 'modern-pdf-lib';

// Decode at half resolution (resolution level 1)
const thumbnail = decodeJpeg2000(j2kBytes, { resolutionLevel: 1 });
// Width and height are halved compared to the full image

// Decode at quarter resolution (resolution level 2)
const preview = decodeJpeg2000(j2kBytes, { resolutionLevel: 2 });
```

Each resolution level halves both dimensions:

| Level | Dimensions | Use Case |
|:------|:-----------|:---------|
| 0 | Full resolution | Final rendering |
| 1 | 1/2 width, 1/2 height | Screen preview |
| 2 | 1/4 width, 1/4 height | Thumbnails |
| 3 | 1/8 width, 1/8 height | Navigation overview |

## Color Spaces

JPEG2000 supports multiple color space representations:

### sRGB

The most common color space for photographic images:

```ts
import { getComponentDepths, summarizeBitDepth } from 'modern-pdf-lib';

const depths = getComponentDepths(j2kBytes);
const info = summarizeBitDepth(depths);

if (info.components === 3 && info.bitsPerComponent === 8) {
  console.log('Standard 8-bit sRGB image');
}
```

### Greyscale

Single-component images (common in medical imaging):

```ts
const depths = getComponentDepths(j2kBytes);
if (depths.length === 1) {
  console.log(`Greyscale image, ${depths[0].bits}-bit`);
}
```

### ICC Profiles

JP2 files can embed ICC color profiles in the `colr` box. When present, the profile is used for accurate color reproduction:

```ts
import { loadPdf } from 'modern-pdf-lib';

// ICC profiles are extracted automatically from JP2 files
// and applied during image rendering within the PDF context.
const pdf = await loadPdf(pdfBytes);
```

### Images with Alpha

JPEG2000 supports alpha channels as additional components. A 4-component image typically represents RGBA:

```ts
const depths = getComponentDepths(j2kBytes);
if (depths.length === 4) {
  console.log('RGBA image with alpha channel');
}
```

## Transcoding to JPEG

For PDFs destined for web or email, you can convert JPEG2000 images to standard JPEG to improve compatibility and reduce file size:

```ts
import { loadPdf, initWasm, optimizeAllImages } from 'modern-pdf-lib';

await initWasm({ jpeg: true });

const pdf = await loadPdf(pdfBytes);
const report = await optimizeAllImages(pdf, {
  quality: 80,
  progressive: true,
});

// JPEG2000 images are decoded, then re-encoded as standard JPEG
console.log(`Transcoded ${report.optimizedImages} images`);
const outputBytes = await pdf.save();
```

> [!TIP]
> Transcoding from JPEG2000 to JPEG is lossy. For archival documents, keep the original JPEG2000 encoding.

## 16-bit Images

Medical and scientific images often use 12-bit or 16-bit component depths. `modern-pdf-lib` provides utilities to work with these:

### Inspecting Bit Depth

```ts
import { getComponentDepths, summarizeBitDepth } from 'modern-pdf-lib';

const depths = getComponentDepths(dicomJ2kBytes);
const info = summarizeBitDepth(depths);

console.log(`${info.components} components, ${info.bitsPerComponent}-bit`);
console.log(`Signed: ${info.isSigned}`);
```

### Converting to 8-bit for Display

```ts
import { downscale16To8 } from 'modern-pdf-lib';

// Convert 16-bit decoded data to 8-bit for display
const data8 = downscale16To8(decodedPixels, 16);
```

### Preserving Full Precision

```ts
import { upscale8To16, normalizeComponentDepth } from 'modern-pdf-lib';

// Upscale 8-bit data to 16-bit for processing
const data16 = upscale8To16(data8);

// Or normalize between any depths (1-16 bits)
const data10to8 = normalizeComponentDepth(data10bit, 10, 8);
```

### Handling Signed Components

Some medical images use signed component values. Offset them to unsigned before display:

```ts
import { offsetSignedToUnsigned, getComponentDepths } from 'modern-pdf-lib';

const depths = getComponentDepths(j2kBytes);
if (depths[0].isSigned) {
  const unsigned = offsetSignedToUnsigned(decodedData, depths[0].bits);
  // Now in the range [0, 2^bits - 1]
}
```

## Performance

### WASM Acceleration

For maximum decoding performance, initialize the JPEG2000 WASM module:

```ts
import { initWasm } from 'modern-pdf-lib';

await initWasm({ jpeg2000: true });
```

When WASM is available, decoding is significantly faster than the pure-JavaScript fallback. If WASM fails to load (e.g. in restricted CSP environments), the JS fallback is used automatically.

### Tiled Decoding

Large JPEG2000 images are often partitioned into tiles. You can decode only the tiles you need:

```ts
import { parseTileInfo, decodeTile, decodeTileRegion } from 'modern-pdf-lib';

const info = parseTileInfo(j2kBytes);
console.log(`${info.tilesX * info.tilesY} tiles total`);

// Decode a specific tile
const tile = decodeTile(j2kBytes, 0);

// Or decode only the region you need
const regionPixels = decodeTileRegion(j2kBytes, {
  x: 100,
  y: 200,
  width: 500,
  height: 400,
});
```

### Tile Assembly

After decoding individual tiles, assemble them into a full image:

```ts
import { parseTileInfo, decodeTile, assembleTiles } from 'modern-pdf-lib';

const info = parseTileInfo(j2kBytes);
const tiles = [];
for (let i = 0; i < info.tilesX * info.tilesY; i++) {
  tiles.push(decodeTile(j2kBytes, i));
}

const fullImage = assembleTiles(tiles, info);
```

### Performance Tips

| Strategy | When to Use |
|:---------|:------------|
| WASM decoding | Always, if available — 5-20x faster |
| Tiled decoding | Large images (>4K) — decode only visible tiles |
| Reduced resolution | Thumbnails, previews — skip wavelet levels |
| Region decoding | Viewport rendering — only decode visible area |
